# 8조 선물상자 - 유어스(You,Us)

![title-image](/public/docs/images/title-image.png)

## 🎁 프로젝트 소개

> **선물은 단순한 소비가 아니라 관계를 표현하는 행위입니다.**
>
> 그러나 대부분의 온라인 쇼핑몰은 카테고리와 가격 필터 중심으로 구성되어 있어, **사용자가 수많은 상품 속에서 스스로 답을 찾아야 하는 구조**입니다.
>
> You, Us는 이 문제를 해결하기 위해 AI 설문을 통해 받는 사람, 상황, 예산 등을 자연어로 입력받고, 이를 분석하여 태그 기반으로 상품을 매칭하는 **선물 추천 쇼핑몰**입니다.
>
> 복잡한 필터를 조합하지 않아도 “20대 여자친구 생일 선물”처럼 **상황 중심으로 탐색**할 수 있으며, 조건에 맞는 상품을 **직관적인 카드 형태로 추천**하는 경험을 제공합니다.
>
> 또한 AI를 단순한 마케팅 요소가 아니라 설계 단계부터 핵심 기능으로 포함하여, 사용자의 자연어 답변을 구조화·태그화함으로써 단순 필터링을 넘어 **맥락 기반 선물 추천 경험을 구현한 프로젝트**입니다.
>
> ---
>
> ### ➡ 우리가 지향하는 가치
>
> - 관계 중심의 커머스
> - 고민을 줄여주는 경험 설계
> - 직관적이고 감성적인 UI
> - 명확한 사용자 흐름과 안정적인 주문 처리

## 🎁 개선사항

### 소혜림

| # | 항목 | 내용 |
| :-: | --- | --- |
| 1 | **fetchClient 리팩토링** | 인증 만료 시 `window.location.href` 대신 `useRouter` 기반 전역 라우터(`router.ts` + `RouterProvider`)를 활용해 새로고침 없이 로그인 페이지로 이동 |
| 2 | **CheckoutItem.product 타입 개선** | `[key: string]: any` 인덱스 시그니처를 제거하고 실제 접근 필드를 명시적 타입으로 교체. `ProductImage`, `ProductExtra`를 `product.types.ts`에서 import해 타입 일원화 |
| 3 | **validateAnswers 유효성 검사 로직 개선** | `zod`를 도입해 입력값 검증 후, 7개 검사 규칙을 `{ test, warning }` 배열로 분리하고 `buildContext`로 조건 플래그를 일원화. 규칙 추가 시 배열에 객체만 추가하면 되는 구조로 개선 |
| 4 | **선물 추천 테스트 - 선택형 UI 개편 및 이전으로 돌아가기** | 텍스트 직접 입력 방식 → 질문별 선택지 제공 방식으로 개편. '기타' 선택 시에만 텍스트 입력 가능. 상단에 진행 단계(예: 3/6)와 이전 버튼을 추가해 이전 답변 수정 가능 |
| 5 | **선물 추천 결과 - h1 태그 추가** | 페이지 제목 `h1` 태그 누락 수정 (`sr-only` 클래스 적용) |
| 6 | **주문/결제 - 페이지 제목 추가** | 장바구니 페이지와 동일한 형태로 '주문・결제' 제목 추가 |
| 7 | **주문/결제 - 모바일 카드 결제 구현** | 모바일 환경에서 결제 창이 팝업 대신 페이지 이동 방식으로 열리는 문제를 해결. 결제 전 주문 데이터를 sessionStorage에 저장하고 `/checkout/mobile-payment` 리다이렉트 페이지에서 결제 성공 여부 확인 후 주문 처리 |
| 8 | **주문/결제 - 중복 주문 상품 합산 처리** | 같은 상품이 장바구니에 분리되어 있을 때 주문 목록에서 수량을 합산해 하나의 항목으로 표시. 주문 상품 수량 표시를 전체 수량 합계가 아닌 상품 종류 수로 변경 |
| 9 | **주문/결제 - 최종 결제 금액 Sticky 처리** | 데스크탑 환경에서 최종 결제 금액 영역이 스크롤에 따라 화면에 고정되도록 sticky 처리 |
| 10 | **주문/결제 - 결제 수단 기본값 제거** | 페이지 진입 시 무통장입금 자동 선택 동작 제거. 사용자가 직접 선택해야 결제 진행되며, 미선택 시 안내 toast 표시 |
| 11 | **주문/결제 - 개인정보 수집 동의 모달 연결** | '개인정보 수집 및 이용 동의' 텍스트 클릭 시 기존 개인정보 처리방침 모달이 열리도록 연결 |
| 12 | **주문/결제 - 카드 결제 테스트 안내 문구 추가** | 카드 결제 선택 시 "카드 결제는 테스트 결제로 진행되며, 자정 무렵 자동 환불됩니다." 안내 문구 표시 |

### 이승규

| # | 항목 | 내용 |
| :-: | --- | --- |
| 1 | **페이지 전용 컴포넌트 기능 단위 분리** | 페이지 종속 컴포넌트를 각 라우트 경로 내 `_components/` 폴더로 이동. 공통 컴포넌트는 `components/ui/`, `components/layout/`, `features/product/`로 역할별 분리해 유지보수성 및 확장성 향상 |
| 2 | **fetchClient 중앙 집중화** | 별도로 API를 호출하던 페이지들을 `fetchClient`로 통일. `requireAuth` param 추가(기본값 `true`, `false`면 인증 로직 스킵) 및 `RequestInit` 타입 활용으로 옵션 지원 |
| 3 | **장바구니 옵션 핸들러 병렬 처리** | 장바구니 옵션 추가 시 `for...of` 순차 호출 → `Promise.all` 병렬 처리로 변경해 네트워크 지연 감축 및 서버 부하 개선 |
| 4 | **장바구니 CartCount 스토리지 분리** | `serverCartCount`가 localStorage에 함께 저장되어 계정 전환 시 이전 값이 노출되는 문제 수정. `partialize`로 비로그인 장바구니(`items`)만 localStorage에 저장하도록 분리 |
| 5 | **폰트 로딩 방식 개선** | `@font-face` → `next/font/local` 방식으로 변경. 빌드 시 `<link rel="preload">` 자동 추가로 폰트 다운로드 시점을 앞당겨 CLS(Cumulative Layout Shift) 감소 |
| 6 | **로딩 컴포넌트 수정** | 공통 Loading 컴포넌트에 2.5초 타이머 추가. 로딩이 2.5초 이상 지속되면 "데이터를 불러오는 중입니다..." 텍스트가 fade-in으로 노출 |
| 7 | **상품 상세 페이지 태그 표출** | 각 상품의 추천 태그를 상세 페이지에 표출. `tagTranslations` 매핑 테이블(`lib/constants/tagTranslations.js`)을 추가해 서버 코드값을 한국어 레이블로 변환하여 표시 |
